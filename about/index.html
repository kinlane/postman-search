About

This is an effort to crowdsource the availability of Corona / COVID-19 virus testing locations arcross all 50 states, using a handful of freely available online tools. Cetralizing the crowdsourcing of testing locations into one data source while using APIs to make sure the data can be syndicated across many locations.

  • Google Sheets - (Sheet) - The core data for this project is storied in a single Google Sheet for each U.S. state, providing a central data source that can be managed by multiple trusted stakeholders.
  • Postman - (Collection) (Docs) - Data is pulled from Google using the Google Sheets API and then updates an API, documentation, and publishes data to GitHub for further display to end users.
  • GitHub - (Repo) - This website is hosted using GitHub using their GitHub pages service, using data originating within Google Sheets, and made available via Postman APIs, to display a listing of locations.

The purpose of This project is to provide a lightweight, easy to use, centralized, and distributed approach to crowdsourcing testing locations as they pop up across 50 states. Using proven services to aggregate, manage, and make virus testing locations easier to find.

If you have questions about how this works, or would like to submit a location you can submit an issue for this project, or email at covid.19.testing.location@gmail.com.

FAQ

These are some of the most common questions that are being asked about this project, make sure to review before submitting a question via issue or email.

  • Why does having an API even matter? - Centrally managing the data keeps it up to date, and having an API allows for real time syndication of the data across as many locations as possible, increasing the reach, while also maintaining the accuracy and freshness of the data.
  • Why are you using Google Sheets? - It provides a free, easy to use medium that a large audience understands for managing the centeralized testing location data.
  • Why are you using Postman? - It provides a free, easy to use medium for publishing and consuming APIs, making the testing location data more accessible to developers.
  • Why are you using GitHub? - It provides a free, easy to use medium that a large audience understands for publishing a static website using the testing location data.
  • Why are there separate sheets for organuzations, locations, physical addresses, phones, and regular schedule? - Because we are using the Open Referral standard which has worked to make data accessible by the widest possible audience, and this approach makes the data available in a spreadsheet format, while also providing a single complete endpoint which stitches it all together.
